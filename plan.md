# Plan - Prototype MVP Musée (inspiré Louvre)

## Vue d'ensemble
Création d'un prototype web MVP pour un musée numérique, inspiré de l'architecture informationnelle du Louvre mais avec un design original supérieur. Deux variantes de design + prototype fonctionnel React + documentation.

## Livrables
- Prototype React + Vite fonctionnel avec routing
- 2 variantes design : Modern/Minimal + Chaleureux/Patrimonial
- Tokens de thème, composants atomiques
- Maquettes visuelles (images générées)
- Guide de style, checklist perf/a11y, README complet

## Stages

### Stage 1 — Analyse & Architecture
- Analyser le site source (https://www.louvre.fr/) pour comprendre l'architecture info
- Définir la structure du prototype (pages, composants, data mocks)
- Lire les skills : vibecoding-webapp-swarm, webapp-building-swarm, web-template-library

### Stage 2 — Design & Assets Visuels
- Définir les tokens de thème pour les 2 variantes
- Générer les assets visuels (images d'œuvres, hero backgrounds)
- Créer les maquettes visuelles (desktop + mobile)

### Stage 3 — Développement Prototype
- Initialiser le projet Vite + React + Tailwind
- Implémenter les composants atomiques (Button, Card, Nav, Footer)
- Implémenter les pages clés : Home, Collections, Fiche Œuvre, Expositions, Visiter
- Implémenter routing, data mocks, animations, interactions
- Générer le contenu FR/EN original

### Stage 4 — Documentation & Audit
- Guide de style condensé
- Checklist performance & accessibilité
- README avec instructions
- Audit Lighthouse

### Stage 5 — Packaging & Déploiement
- ZIP complet du projet
- Déploiement du prototype
- Document synthétique final

## Skills à charger
- Stage 1 : webapp-building-swarm (tools), vibecoding-webapp-swarm (workflow)
- Stage 3 : vibecoding-webapp-swarm (dev workflow)
- Stage 5 : deploy
